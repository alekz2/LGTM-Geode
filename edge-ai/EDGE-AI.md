# Geode Edge-AI — Anomaly Detection with ONNX

Adds ML-based anomaly detection to the LGTM-Geode observability lab. An IsolationForest model is trained offline on historical Geode metrics, exported to ONNX, and deployed as a lightweight Python inference service in the existing Docker Compose on BlackWidow. Anomaly scores are exposed as Prometheus metrics, scraped by Alloy, stored in Mimir, and visualized in Grafana.

## Architecture

```
Geode JVM (Antman / Hulk)
  └── JMX Exporter
        └── Alloy ──────────────────────────────► Mimir
                                                     │
                                    query last 5min  │
                                    ◄────────────────┘
                               geode-ai service (BlackWidow:9500)
                                    │
                          anomaly scores /metrics
                                    │
                               Alloy scrape ────────► Mimir ──► Grafana
```

## What gets scored

The model evaluates **server1** (Antman) and **server2** (Hulk) only. Locators are excluded — they have a different metric profile from data servers.

Ten features are computed per member from the last 5 minutes of Mimir data:

| # | Feature | Source |
|---|---------|--------|
| 0 | `heap_used_ratio` | JVM heap used / max |
| 1 | `gc_pause_rate` | Rate of GC pause seconds (all GC types) |
| 2 | `gc_count_rate` | Rate of GC collections (all GC types) |
| 3 | `gets_rate` | `gemfire_member_getsrate` |
| 4 | `puts_rate` | `gemfire_member_putsrate` |
| 5 | `get_latency_ns` | `gemfire_cacheserver_getrequestavglatency` |
| 6 | `put_latency_ns` | `gemfire_cacheserver_putrequestavglatency` |
| 7 | `cpu_usage` | `gemfire_member_cpuusage` |
| 8 | `get_request_rate` | `gemfire_cacheserver_getrequestrate` |
| 9 | `put_request_rate` | `gemfire_cacheserver_putrequestrate` |

Feature definitions live in [`features.json`](features.json). Both `train.py` and the inference service load this file — it is the single source of truth for feature order.

## Prometheus metrics exposed

| Metric | Type | Description |
|--------|------|-------------|
| `geode_anomaly_score{instance, member}` | Gauge | IsolationForest score — more negative = more anomalous |
| `geode_ai_last_inference_timestamp_seconds{member}` | Gauge | Unix timestamp of last successful inference |
| `geode_ai_inference_errors_total{member, reason}` | Counter | Errors during Mimir query or ONNX inference |

**Score interpretation:** IsolationForest `decision_function` output. Healthy operation typically produces scores near 0 or slightly positive. Scores below **-0.05** (default alert threshold) indicate anomalous behaviour — tune this after observing your cluster's baseline.

## Step 1 — Train the model (Ironman)

> Run once. Re-run any time you want to retrain on fresher data.

```cmd
cd D:\Alex\Work\Installs\LGTM-Geode\edge-ai\training

python -m venv venv
venv\Scripts\activate

pip install -r requirements.txt

python train.py ^
  --mimir-url http://192.168.0.153:9009/prometheus ^
  --output-dir models ^
  --lookback-hours 24
```

The script will print a score distribution table when it finishes:

```
Score distribution (decision_function on training data):
  min=-0.1823  5th%=-0.0812  mean=0.0241  95th%=0.0934  max=0.1701
  Suggested alert threshold: -0.0812 (5th percentile of training data)
```

Use the **5th percentile** value as your Grafana alert threshold — it is the score below which only 5% of normal training data falls.

### Optional flags

| Flag | Default | Description |
|------|---------|-------------|
| `--lookback-hours` | `24` | Hours of history to train on — use more for a better baseline |
| `--contamination` | `0.05` | Expected fraction of anomalies in training data — lower if your cluster was fully healthy during training |
| `--step` | `60` | Query step in seconds |

### Validate the ONNX file

```cmd
python -c "import onnxruntime as rt, numpy as np; s=rt.InferenceSession('models/geode_anomaly.onnx'); print(s.run(None, {'float_input': np.zeros((1,10), dtype='float32')}))"
```

Expected output: `[array([-1]), array([<score>])]`

## Step 4 — Add the service to Docker Compose (BlackWidow)

Open `/home/alex/observability-lgtm/docker-compose.yml` and add the `geode-ai` service block from [`docker-compose-geode-ai.yml`](docker-compose-geode-ai.yml) into the `services:` section.

The key sections to add:

```yaml
  geode-ai:
    image: geode-ai:latest
    container_name: geode-ai
    restart: unless-stopped
    ports:
      - "9500:9500"
    volumes:
      - /home/alex/observability-lgtm/edge-ai/models:/models:ro
      - /home/alex/observability-lgtm/edge-ai/features.json:/features.json:ro
    environment:
      GEODE_AI_MIMIR_URL: "http://mimir:9009/prometheus"
      GEODE_AI_MODEL_PATH: "/models/geode_anomaly.onnx"
      GEODE_AI_POLL_INTERVAL: "60"
      GEODE_AI_LOOKBACK_MINUTES: "5"
      GEODE_AI_MEMBERS: "server1:antman,server2:hulk"
    depends_on:
      - mimir
```

Then start it:

```bash
cd /home/alex/observability-lgtm
docker compose up -d geode-ai
docker compose logs geode-ai
```

### Verify the service is healthy

```bash
curl http://localhost:9500/health
```

Expected response (HTTP 200):

```json
{
  "status": "ok",
  "model_loaded": true,
  "members": ["server1", "server2"],
  "last_inference_age_seconds": {"server1": 42.3, "server2": 43.1}
}
```

If `model_loaded` is `false`, the ONNX file is missing or unreadable at `/models/geode_anomaly.onnx`. If `status` is `"degraded"`, the service is running but no inference has completed yet (wait one poll interval — default 60s).

```bash
# Confirm metrics are emitting
curl -s http://localhost:9500/metrics | grep geode_anomaly_score
```

Expected:

```
geode_anomaly_score{instance="antman",member="server1"} -0.0231
geode_anomaly_score{instance="hulk",member="server2"} -0.0187
```

## Step 6 — Create Grafana dashboard

Create a new dashboard named **Geode Anomaly Detection** with these panels:

### Dashboard variable

| Setting | Value |
|---------|-------|
| Name | `member` |
| Type | Query |
| Query | `label_values(geode_anomaly_score, member)` |
| Multi-value | Yes |
| Default | `.*` |

### Panels

**Member Anomaly Score** — Time series
```promql
geode_anomaly_score{member=~"$member"}
```
Legend: `{{instance}} / {{member}}`. Set Y-axis min `-0.5`, max `0.5`. Add a threshold line at your calibrated value (e.g. `-0.05`).

**Anomaly Status** — Stat panel
```promql
geode_anomaly_score{member=~"$member"}
```
Calculation: Last. Thresholds: base = Red, `-0.05` = Green (scores above threshold are green).

**Inference Freshness** — Stat panel
```promql
time() - geode_ai_last_inference_timestamp_seconds{member=~"$member"}
```
Unit: seconds. Thresholds: Green < 180s, Yellow 180–300s, Red > 300s.

**Inference Errors** — Time series
```promql
rate(geode_ai_inference_errors_total{member=~"$member"}[5m])
```

### Alert rule

Create under Alerting > Alert Rules > New alert rule:

| Setting | Value |
|---------|-------|
| Name | `Geode Member Anomaly Detected` |
| Query | `geode_anomaly_score` |
| Condition | last() IS BELOW `-0.05` |
| Evaluate every | `1m` |
| Fire after | `2m` |
| Summary | `Geode member {{ $labels.member }} on {{ $labels.instance }} is anomalous` |
| Description | `Score {{ $values.A.Value \| printf "%.4f" }} — investigate heap, GC, or latency` |

## Threshold calibration

The default alert threshold of `-0.05` is a starting point. Calibrate it after deployment:

1. Let the cluster run normally for 30 minutes after deployment.
2. In Grafana Explore, run: `geode_anomaly_score` and note the steady-state range.
3. Run StressTestApp on Ironman (`run_stress.bat`) and confirm the score drops noticeably under load.
4. Set the threshold just below the lowest score seen during normal operation.

The training script also prints the 5th-percentile of training data as a suggested threshold — use that as a first estimate.

## Troubleshooting

**`model_loaded: false` in /health**
The container cannot find `/models/geode_anomaly.onnx`. Check that the bind-mount path exists on the host: `ls /home/alex/observability-lgtm/edge-ai/models/`

**`status: degraded` in /health immediately after start**
Normal — wait one poll interval (60s default) for the first inference to complete.

**`geode_anomaly_score` missing from /metrics**
Check container logs for Mimir query errors: `docker compose logs geode-ai`. Ensure Geode is running and metrics are visible in Grafana before inference will produce data.

**`up{job="geode-ai"}` is 0 in Mimir**
Alloy cannot reach the `geode-ai` container. Confirm both are on the same Docker network: `docker network inspect observability-lgtm_default`.

**Score never goes negative under stress**
The contamination parameter was too high during training, or the StressTestApp load is not strong enough to push metrics outside the training distribution. Retrain with `--contamination 0.01` or run a longer / heavier stress test while observing raw metrics.
